# Audit UX — Mycelium Fleet OS
Date : 2026-06-07

---

## Résumé exécutif

L'application dispose d'une base de composants UI cohérente (design system Mycelium, Tailwind v4, couleurs oklch) mais souffre de lacunes critiques sur la gestion des états : aucune page ne propose de squelette de chargement fonctionnel branché sur les vraies requêtes Convex, et les états vides sont soit absents soit non stylisés. Le layout authentifié (`authenticated-layout.svelte`) est solide, mais la sidebar ne distingue pas visuellement les deux environnements (admin vs salarié) par leur couleur d'accent respective. Plusieurs composants métier (VehicleCard, ReservationCalendar, ChatContainer) contiennent des pratiques inaccessibles : boutons sans `aria-label`, icônes cliquables sans texte alternatif, contrastes de texte non conformes WCAG AA. La page marketing est le point le plus urgent : elle mélange des classes Tailwind ad-hoc avec des valeurs de couleur hardcodées en dehors du design system, sans aucun état de chargement ni H1 sémantique.

---

## Palette de priorités

- **P0** : Bloquant UX (utilisateur bloqué, données illisibles, inaccessible, non conforme WCAG AA)
- **P1** : Dégradé significatif (état manquant fréquent, incohérence flagrante avec le design system)
- **P2** : Amélioration (polish, cohérence fine, animations, micro-interactions)

## Page marketing — `/`
Fichiers : `src/routes/[[lang]]/(marketing)/+page.svelte`, `src/blocks/hero/hero-four.svelte`

### P0
- [ ] **Pas de `<h1>` unique et sémantique** : le fichier `+page.svelte` contient plusieurs `<h2>` et titres visuels sans `<h1>` parent. Les lecteurs d'écran ne peuvent pas identifier le titre principal de la page.
- [ ] **Couleurs hardcodées hors design system** : `hero-four.svelte` utilise `text-gray-900`, `bg-white`, `text-blue-600` au lieu des variables oklch définies dans `app.css`. En mode dark, le contraste tombe sous 4.5:1 (violation WCAG 1.4.3).
- [ ] **Absence de balise `lang` garantie** : le layout `[[lang]]` est optionnel ; sans paramètre de route, l'attribut `lang` sur `<html>` n'est pas positionné automatiquement, rendant la page non conforme WCAG 3.1.1.

### P1
- [ ] **Touch targets insuffisants sur mobile** : les liens CTA dans `hero-four.svelte` utilisent `py-2 px-4`, générant des cibles tactiles ~36px — en dessous du seuil de 44px recommandé.
- [ ] **Pas de transitions de navigation** : aucun `onNavigate` configuré dans `src/routes/+layout.svelte` pour une animation de sortie entre la landing et les routes auth.
- [ ] **Pas d'indicateur de chargement** : aucun feedback visuel lors du clic sur les CTA vers les routes authentifiées.

### P2
- [ ] **Typographie incohérente** : `hero-four.svelte` mélange `font-sans` (Tailwind default) et `font-geist`. Aucune classe `font-mono` pour les données chiffrées (prix, statistiques).
- [ ] **Pas de `<meta name="description">` dans le layout marketing** : le `+layout.svelte` marketing ne surcharge pas `<svelte:head>` avec une description SEO.

## Dashboard Admin — `/admin/dashboard`
Fichiers : `src/routes/[[lang]]/admin/dashboard/+page.svelte`, `src/lib/components/dashboard/stat-card.svelte`

### P0
- [ ] **StatCard : tendance communiquée par couleur seule** : la flèche ↑/↓ et sa couleur (vert/rouge) sont le seul signal de tendance positive/négative. Aucun `aria-label` ne communique la signification aux lecteurs d'écran (violation WCAG 1.4.1).
- [ ] **Couleur d'accent admin incorrecte** : le dashboard utilise `text-blue-500` (bleu salarié) au lieu de la couleur admin du design system sur les éléments interactifs principaux.

### P1
- [ ] **Skeleton non branché sur Convex** : `+page.svelte` n'utilise pas `$derived` pour conditionner l'affichage d'un `<Skeleton>` pendant l'état `undefined` de la query. Les StatCards affichent `undefined` dans le DOM.
- [ ] **Empty state absent** : si l'organisation n'a aucun véhicule ni réservation, le dashboard affiche des `0` sans CTA contextuel ("Importer votre première flotte →").
- [ ] **Grid non responsive sur mobile** : `grid-cols-4` sans breakpoint `sm:grid-cols-2` ni `grid-cols-1`. Overflow horizontal sur mobile (<640px).

### P2
- [ ] **Pas d'animation d'entrée en cascade** sur les StatCards (stagger 50ms fade-in attendu).
- [ ] **Pas de tooltip** sur les métriques expliquant la formule de calcul (ex : "Taux d'utilisation = réservations actives / total véhicules").

## Page salarié (accueil) — `/app`
Fichiers : `src/routes/[[lang]]/app/+page.svelte`, `src/lib/components/chat/chat-container.svelte`, `src/lib/components/chat/chat-input.svelte`, `src/lib/components/calendar/reservation-calendar.svelte`

### P0
- [ ] **Bouton d'envoi dans `chat-container.svelte` sans `aria-label`** : `<button>` avec icône `<SendIcon />` seule, sans texte accessible. Inaccessible au clavier et aux lecteurs d'écran.
- [ ] **Pas de `aria-live="polite"` sur le conteneur de messages** : les réponses IA apparaissent sans notification aux lecteurs d'écran (violation WCAG 4.1.3).
- [ ] **Couleur d'accent salarié absente** : `/app/+page.svelte` utilise la couleur par défaut Button (vert admin) au lieu de la couleur salarié. Confusion environnementale.

### P1
- [ ] **Empty state du chat absent** : `chat-container.svelte` n'affiche aucun état initial accueillant (ex : suggestions de requêtes "Réservez un véhicule pour demain…").
- [ ] **État d'erreur réseau non géré** : si la query Convex ou l'action Claude API échoue, aucun message d'erreur n'est affiché.
- [ ] **`ReservationCalendar` overflow horizontal** : structure de 7 colonnes sans `overflow-x-auto` parent. Scroll horizontal sur toute la page sur mobile.
- [ ] **Pas de skeleton sur le calendrier** : grille de jours vide pendant 300-500ms au chargement.

### P2
- [ ] **Transitions de message absentes** : les nouveaux messages s'insèrent sans animation `slide-in-up`.
- [ ] **`textarea` dans `chat-input.svelte` non auto-extensible** : hauteur fixe `h-12`, scroll interne pour les messages longs.

## Paramètres salarié — `/app/settings`
Fichier : `src/routes/[[lang]]/app/settings/+page.svelte`

### P0
- [ ] **Inputs sans `<label>` associé** : plusieurs champs utilisent `placeholder` comme seul identifiant, sans `<label for="...">` correspondant (violation WCAG 1.3.1).
- [ ] **Focus invisible sur les inputs** : `input.svelte` applique `outline-none` sans substitut (`ring`, `focus-visible:outline`). Violation WCAG 2.4.7.

### P1
- [ ] **Pas de feedback après soumission** : aucun toast succès/erreur après sauvegarde des paramètres (contrairement à `/admin/settings/organization`).
- [ ] **Bouton de sauvegarde non désactivé pendant la mutation** : double-clic possible, risque de soumissions multiples.
- [ ] **Section "Préférences de notification" absente** : référencée dans `app-sidebar-config.ts` mais pas implémentée dans la page.

### P2
- [ ] **Labels utilisent `text-gray-500`** au lieu de la variable CSS `--color-muted` du design system.

## Paramètres organisation admin — `/admin/settings/organization`
Fichier : `src/routes/[[lang]]/admin/settings/organization/+page.svelte`

### P0
- [ ] **Select "Secteur" et "Taille" sans association `label`/`id`** : le composant `select.svelte` custom ne propage pas correctement l'`id` ciblé par le `<label for="...">`. Les lecteurs d'écran ne lient pas le label au champ.
- [ ] **Pas de gestion UI de l'état "accès refusé"** : si le redirect server-side `ORG_ADMIN` échoue, la page s'affiche vide sans message d'erreur ni redirection visible.

### P1
- [ ] **Validation SIREN côté client absente** : le champ accepte n'importe quelle chaîne, l'erreur de format arrive après la mutation (délai réseau).
- [ ] **Bouton "Enregistrer" sans état loading** : double-clic possible pendant la mutation `updateOrganization`.
- [ ] **Pas de confirmation sur changement de nom d'organisation** : action critique sans dialog de confirmation.

### P2
- [ ] **Pas de breadcrumb** : la page est à 3 niveaux de profondeur sans fil d'Ariane. Le composant `page-header.svelte` supporte la prop `breadcrumbs` mais elle n'est pas passée.
- [ ] **Toast de succès trop court** : 2 secondes (défaut) est insuffisant pour une action aussi critique — 4 secondes recommandées.

### P0

- [ ] **Bouton de profil dans `authenticated-header.svelte` sans `aria-label`** : bouton avatar sans texte accessible.
- [ ] **Pas de focus trap dans le menu utilisateur** : le focus peut sortir du dropdown sans le fermer (violation WCAG 2.1.2).

### P1
- [ ] **Sidebar mobile non fermable avec Escape** : l'overlay de fermeture n'a pas de gestionnaire `keydown` pour `Escape`.
- [ ] **`sidebar-trigger.svelte` sans `aria-expanded`** : le bouton hamburger ne communique pas l'état ouvert/fermé.
- [ ] **Organisation switcher sans état loading** pendant le changement de contexte.

### P2
- [ ] **Transition de fermeture de sidebar absente** : ouverture animée, fermeture instantanée.

## Composants UI de base

### button.svelte — `src/lib/components/ui/button/button.svelte`

- [ ] **Prop `loading` absente** : pas de spinner intégré + blocage d'interaction. Chaque page reimplemente sa propre logique de manière incohérente.

### input.svelte — `src/lib/components/ui/input/input.svelte`

#### P0

## Problèmes transversaux

### P0
- [ ] **Aucune page n'a de skip-link** : `authenticated-layout.svelte` et `+layout.svelte` racine n'incluent pas `<a href="#main-content" class="sr-only focus:not-sr-only">Passer au contenu</a>`.
- [ ] **`dialog.svelte` et `select.svelte` non conformes ARIA** : utilisés dans plusieurs pages, l'impact est transversal.
- [ ] **Police Geist non chargée de manière garantie** : `app.css` référence `font-family: 'Geist Sans'` sans `@font-face` ni import Fontsource visible dans `+layout.svelte`. Fallback silencieux `sans-serif` en production.

### P1
- [ ] **Couleurs d'accent par environnement non implémentées** : les composants partagés utilisent une seule couleur (vert admin). Le layout `/app/+layout.svelte` n'override pas les variables CSS pour le contexte salarié.
- [ ] **Gestion d'erreur Convex non standardisée** : chaque page gère (ou ignore) les erreurs différemment. Pas de composant `<ErrorBoundary>` ni de pattern `$effect` cohérent.
- [ ] **Pas de composant `<EmptyState>` partagé** : dashboard, fleet-table, chat et calendar recréent chacun le pattern empty state. Priorité haute — 1 composant, impact sur 5+ écrans.
- [ ] **Overflow horizontal mobile non corrigé** : `fleet-table.svelte` et `reservation-calendar.svelte` n'ont pas de `overflow-x-auto` wrapper.

### P2
- [ ] **Pas de page `+error.svelte`** : les erreurs 404/500 affichent le message brut SvelteKit sans mise en forme Mycelium.
- [ ] **Pas de transition entre routes** : `onNavigate` + `document.startViewTransition` non configuré dans le layout racine.
- [ ] **Toast sans prop `duration` configurable** : durée fixe inadaptée selon la criticité du message.
- [ ] **Pas de composant `<LoadingSpinner>` standardisé** : chaque composant crée son propre `animate-spin` inline.
